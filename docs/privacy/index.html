Privacy Policy

Restaurant POS + MDM (Demo)  ·  Android package by.vsdev.posterminal.demo

Last updated: 14 July 2026

This application is a portfolio / demonstration app. It does not process real payments, real customer orders, or real financial data. The "payment" step is a non-functional stub.

This Privacy Policy explains what information the Restaurant POS + MDM (Demo) application ("the App") handles, why, and your choices regarding it. By installing and using the App you agree to the practices described here.

Information the App collects

The App is a demonstration of a point-of-sale terminal with a mobile-device-management (MDM) agent. To show remote device management, each device sends a small amount of non-personal technical data to the demo backend:

  • Device identifier — a random value (e.g. pos-1a2b3c4d) generated on the device the first time the App runs. It is not derived from any hardware identifier (no IMEI, no Android ID, no advertising ID) and cannot be used to identify you.
  • Device name — a label you type yourself during enrollment.
  • Device model — the public model string (e.g. "Pixel 7"), used for display in the admin console.
  • Battery level and basic status flags (online/offline, kiosk mode, payment restriction) and a timestamp, sent as periodic "heartbeats".
  • Enrollment token — an optional code scanned from a QR code to associate the device with a demo account.

Information the App does NOT collect

  • No name, email, phone number, or any personal contact information.
  • No location data.
  • No contacts, photos, files, microphone, or camera data (the camera, if used, only scans an enrollment QR code locally and stores no images).
  • No real payment or card data — payments are simulated.
  • No advertising identifiers, and no third-party advertising or analytics SDKs.

How the information is used

Collected technical data is used solely to demonstrate MDM features: listing devices, showing their live status, and sending remote commands (e.g. lock, enable kiosk mode) from the web admin console. It is never sold, rented, or used for advertising or profiling.

Permissions

The App requests only the INTERNET permission, which is required to communicate with the demo backend.

Where data is stored and third parties

The demo backend and admin console are hosted on third-party infrastructure purely for demonstration:

  • Render (backend API and database) — render.com/privacy
  • Vercel (web admin console hosting) — vercel.com/legal/privacy-policy

No other third parties receive data from the App.

Data retention and deletion

Device records are kept only to demonstrate the console. A device (and all of its data) can be deleted at any time from the admin console, which removes it from the backend. Because this is a demo, records may also be cleared periodically without notice. Uninstalling the App removes all data stored locally on the device.

Children's privacy

The App is intended for demonstration to a general/professional audience and is not directed at children under 13. It does not knowingly collect any information from children.
